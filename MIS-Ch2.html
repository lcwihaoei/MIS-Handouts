                                                                                  • Ch2 講義：從流程到洞察——資訊系統如何驅動企業卓越營運

                                                                                  拆解現代企業的數位化核心競爭力（重點整理版）

                                                                                  本章核心一句話

                                                                                  企業要「跑得快、做得準、改得動」，關鍵在於：

                                                                                  1. 把商業流程拆清楚（流程）

                                                                                  2. 用資訊系統把流程做順做強（系統）

                                                                                  3. 用協同合作把知識流動起來（協作/社群）

                                                                                  4. 用治理確保IT投資對準策略並創造價值（治理）

                                                                                  全球性企業的共同挑戰：協作與創新遇到阻礙（案例：Sanofi Pasteur）

                                                                                  過去的困境（典型問題）

                                                                                  • 缺乏合適工具鼓勵員工跨部門對話與分享想法

                                                                                  • 階層式「由上而下」流程＋員工分布各地 → 形成 知識孤島（Information Silos）

                                                                                  • 創新與品質改善潛力受限於組織結構與溝通方式

                                                                                  分析切入：管理 × 組織 × 科技 三個維度

                                                                                  管理（Management）

                                                                                  • 需要發展「知識分享」的策略與目標

                                                                                  組織（Organization）

                                                                                  • 需要重新設計知識分享與協同合作流程

                                                                                  • 需要改變組織文化

                                                                                  科技（Technology）

                                                                                  • 導入協作工具（例：Microsoft Yammer / 企業社群網路）

                                                                                  從挑戰到解法的邏輯鏈

                                                                                  • 企業挑戰：階層式流程＋員工分散全球

                                                                                  • 資訊系統：企業社群網路 → 提供知識獲取、創新與協作的新管道

                                                                                  • 企業解決方案：改善生產力、降低成本、增加創新、強化品質

                                                                                  企業營運的核心：解構商業流程（Business Processes）

                                                                                  定義（Key Definitions）

                                                                                  商業流程：具備組織化與協調性，並聚焦於生產有價值的產品/服務的工作方式。

                                                                                  不只是步驟，更是企業的知識、經驗與核心能力的體現。

                                                                                  核心概念（Core Idea）

                                                                                  • 每個企業都可視為「商業流程的集合體」

                                                                                  • 企業績效直接取決於流程設計與協調的優劣

                                                                                  三個關鍵詞（常考）

                                                                                  • Value（價值）

                                                                                  • Product/Service（產品/服務）

                                                                                  • Coordination（協調）

                                                                                  流程貫穿每個功能領域（Functional Areas）

                                                                                  製造與生產

                                                                                  • 組裝產品、檢查品質、產生物料單

                                                                                  銷售與行銷

                                                                                  • 定義客戶、讓客戶意識到產品、銷售產品

                                                                                  財務與會計

                                                                                  • 支付債權人、編製財務報表、管理現金帳戶

                                                                                  人力資源

                                                                                  • 招聘員工、評估績效、提供退休金計畫

                                                                                  關鍵流程成功：跨部門無縫協作（例：訂單履約流程）

                                                                                  Order Fulfillment Process（概念）

                                                                                  跨部門串接才能完成一筆訂單：

                                                                                  • 銷售：產生訂單 → 送出訂單

                                                                                  • 會計：信用稽核 → 信用核準 → 產生發票

                                                                                  • 生產：產品組裝 → 產品交運

                                                                                  本質：複雜程序，需要銷售/會計/生產等功能緊密協同合作。

                                                                                  資訊科技如何「優化並革新」商業流程（必背三點）

                                                                                  自動化（Automation）

                                                                                  • 把過去手動的流程步驟自動化 → 提升效率與準確性

                                                                                  資訊透明化（Information Flow）

                                                                                  • 改變資訊流動方式 → 讓更多人即時存取與分享

                                                                                  • 將傳統「循序處理」轉為更高效的「同步進行」

                                                                                  模式創新（New Business Models）

                                                                                  • 支援並催生全新營運模式 → 徹底改變價值創造方式

                                                                                  企業內的資訊系統：賦能決策的引擎

                                                                                  本章用兩個代表系統來理解「從作業到決策」：

                                                                                  • TPS（交易處理系統）：營運基礎

                                                                                  • MIS（管理資訊系統）：從數據到決策（也可視為BI落地工具之一）

                                                                                  TPS：營運的基石——交易處理系統（Transaction Processing System）

                                                                                  定義

                                                                                  TPS 是電腦化系統，負責執行並記錄企業日常交易。

                                                                                  核心特性（Key Characteristics）

                                                                                  • 層級：作業層級（Operational Level）

                                                                                  • 功能：回答例行性問題、追蹤組織內交易流程

                                                                                  • 要求：資訊必須容易取得、即時更新且高度準確

                                                                                  （投影片例：薪資系統輸出薪資、管理報表、送政府機構、送總帳）

                                                                                  MIS：從數據到決策——管理資訊系統（Management Information Systems）

                                                                                  定義

                                                                                  MIS：提供中階管理者組織目前績效的報告，用以監控企業與預測未來績效。

                                                                                  MIS 與 BI（Business Intelligence）

                                                                                  • MIS 是實現「企業智慧（BI）」的關鍵工具

                                                                                  • BI：透過資料與軟體工具，組織與分析企業資料，協助制定更明智決策

                                                                                  核心功能（Core Function）（很常考）

                                                                                  • MIS 將來自 TPS 的大量原始交易數據
                                                                                    → 轉換為管理者能理解與使用的「摘要資訊」

                                                                                  系統協同運作：TPS 如何支援 MIS（圖必懂）

                                                                                  邏輯

                                                                                  多個 TPS 產生交易資料（訂單、MRP、總帳…）
                                                                                  → 形成 MIS 檔案（銷售、成本、變更、費用…）
                                                                                  → MIS 產出：

                                                                                  • 報告（Reports）

                                                                                  • 線上呈現與儀表板（Online Displays & Dashboards）
                                                                                    → 管理者取得宏觀營運視圖

                                                                                  典範轉移：從「協同合作」到「社群企業」

                                                                                  協同合作（Collaboration）

                                                                                  • 與他人一同工作以達成共享且明確的目標

                                                                                  社群企業（Social Business）

                                                                                  • 不只內部合作

                                                                                  • 使用社群工具（如臉書、推特、企業內部平台）深化員工、客戶、供應商之間的全面互動

                                                                                  社群企業本質關鍵詞

                                                                                  • 對話（Dialogue）：建立並促進有意義的對話，打破組織內外藩籬

                                                                                  協同合作的商業效益：可衡量的價值主張（常考整理）

                                                                                  • 生產力（Productivity）：更快解決問題、降低錯誤率、獲得專業知識

                                                                                  • 品質（Quality）：更快發現/修正錯誤，減少設計與生產拖延

                                                                                  • 創新（Innovation）：打破部門壁壘，催生更多元產品/服務與管理創新（群眾智慧）

                                                                                  • 客戶服務（Customer Service）：更快、更有效率解決客訴

                                                                                  • 財務績效（Financial Performance）：卓越公司在銷售/成長/財務績效更佳

                                                                                  結論：協同合作的公司更可能擁有卓越的銷售、成長與財務績效。

                                                                                  深入：企業社群網路（Enterprise Social Network）的核心功能

                                                                                  企業社群網路：將社群力量導入組織內部的「神經中樞」

                                                                                  • Profiles（輪廓檔案）：建立成員檔案，描述技能、專案、學經歷、興趣

                                                                                  • Groups & Team Workspaces（團體與團隊工作坊）：建立群組分享資訊、協作文件、儲存對話以保存知識

                                                                                  • Content Sharing（內容分享）：分享/儲存/管理文件、簡報、影像、視訊

                                                                                  • Feeds & Notifications（匯入與通知）：即時資訊串流、狀態更新、公告

                                                                                  • Tagging & Social Bookmarking（貼標籤與社群書籤）：標註、喜歡、加關鍵字、便於搜尋與追蹤

                                                                                  • Permissions & Privacy（許可與隱私）：確保資訊在設定的可見範圍內流通

                                                                                  社群企業的多元應用場景（八大類）

                                                                                  • Communities（社群）：公開論壇討論議題、分享專業知識

                                                                                  • Social Networks（社群網路）：透過個人/企業輪廓資料串連

                                                                                  • Crowdsourcing（群眾資源）：匯聚知識產生新想法與答案

                                                                                  • Shared Workspace（共享工作空間）：協調專案與任務，共同創造內容

                                                                                  • Blogs & Wikis（部落格與維基）：發布並迅速存取知識，討論想法與經驗

                                                                                  • Social Commerce（社群商務）：在社群平台分享購買/採購意見

                                                                                  • File Sharing（檔案分享）：上傳、分享與評論照片/影像/聲音/文字

                                                                                  • Social Marketing（社群行銷）：與客戶互動，取得客戶見解

                                                                                  協同合作與社群企業的科技工具箱（考題會混）

                                                                                  • Email & Instant Messaging（電子郵件與即時傳訊）

                                                                                  • Wikis（維基）

                                                                                  • Virtual Meeting Systems（虛擬會議系統）

                                                                                  • Cloud Collaboration Services（雲端協同合作服務：Google Drive、Microsoft 365）

                                                                                  • Enterprise Social Networking Tools（企業社群連網工具）

                                                                                  • Collaboration & Social Business Platforms（協作/社群平台：IBM Notes、Microsoft Teams、Slack）

                                                                                  選工具的方法：時間 × 空間（Time & Place）協作矩陣（必背四格）

                                                                                  依「同處/遠端」與「同步/非同步」分類

                                                                                  同處 × 同步：面對面溝通

                                                                                  • 決策室、共享桌面、壁掛螢幕、牆壁投影等

                                                                                  同處 × 非同步：連續型任務

                                                                                  • 團隊室、大型公眾顯示器、輪班群組軟體、專案管理等

                                                                                  遠端 × 同步：遠端互動

                                                                                  • 視訊會議、即時訊息、螢幕共享、虛擬世界、多使用者編輯等

                                                                                  遠端 × 非同步：溝通 + 協調

                                                                                  • 電子郵件、布告欄、部落格、非同步會議、維基等

                                                                                  策略性領導：資訊科技治理（IT Governance）與關鍵角色

                                                                                  IT Governance 定義

                                                                                  • 探討組織使用資訊科技的策略與政策

                                                                                  目的（Purpose）（兩點）

                                                                                  • 規範決策權與責任歸屬的架構（Framework）

                                                                                  • 確保資訊科技能支援組織的策略與目標（Alignment） -（延伸強調）確保投資能創造最大價值（Value Maximization）

                                                                                  關鍵領導角色（常考縮寫）

                                                                                  • CIO（資訊長）

                                                                                  • CSO / CISO（安全長 / 資安長）

                                                                                  • CPO（隱私總管）

                                                                                  • CKO（知識長）

                                                                                  • CDO（資料長）

                                                                                  考前速記（超精簡）

                                                                                  • 商業流程 = 組織化 + 協調 + 創造價值（產品/服務）

                                                                                  • IT 改善流程三件事：自動化、資訊透明化、模式創新

                                                                                  • TPS：作業層、記錄日常交易、即時與準確

                                                                                  • MIS：中階管理、報告監控與預測、把TPS大量資料→摘要資訊

                                                                                  • Collaboration vs Social Business：目標協作 vs 擴展到員工/客戶/供應商互動與對話

                                                                                  • 協作矩陣四格：同處/遠端 × 同步/非同步

                                                                                  • IT Governance：Framework + Alignment + Value
